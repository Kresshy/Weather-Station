# Weather Station v3.6.7 - The Driver Persistence Update

## 🛠️ Bug Fixes & Refinements
*   **Unified Connection Plane Persistence**: Resolved a critical issue where BLE devices were incorrectly routed to the Classic Bluetooth driver during reconnection attempts. 
*   **Android "Type Flip" Mitigation**: Implemented a persistence layer in `CompositeConnection` that remembers the successful driver (BLE vs. Classic) for a specific hardware address. This prevents the system from defaulting to Classic when Android reports an `UNKNOWN` (0) device type immediately after a link loss.
*   **Session Continuity**: Ensured that the `currentDevice` address is used to validate that any "driver guess" for an unknown device type honors the existing active session, preventing redundant stop/start cycles.

## 🧪 Quality & Tests
*   **Regression Tests**: Added a new test case to `CompositeConnectionTest.java` that specifically reproduces the "BLE to UNKNOWN" type flip.
*   **Full Suite Verification**: All 100+ unit tests passed, confirming that the analytical heartbeat and data parsing remain unaffected by the connectivity routing fix.
*   **Build Integrity**: Verified through the mandatory sequence: `./gradlew spotlessApply test build`.
