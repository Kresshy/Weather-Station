# Weather Station v3.6.6 - The Reconnection Resilience Update

## 🛠️ Bug Fixes & Refinements
*   **Unified Reconnection Plane**: Fixed a critical bug in `CompositeConnection` that blocked reconnection attempts when a device was in the `disconnected` state. The router now correctly permits retries while still protecting against redundant "connecting" or "connected" calls.
*   **BLE Resource Hardening**: Implemented explicit `gatt.close()` calls in `BleConnection` upon detecting a link loss. This prevents GATT resource exhaustion and ensures the Android Bluetooth stack remains stable over multiple reconnection cycles.
*   **GATT Lifecycle Safety**: Added a preemptive "Disconnect & Close" check in `BleConnection.connect()` to ensure any stale or partially open connections are fully purged before a new handshake begins.
*   **Seamless Hot-Swapping**: Verified that switching between Classic Bluetooth and BLE hardware (and vice-versa) correctly stops the active driver and releases all hardware resources before starting the new one.

## 🧪 Quality & Tests
*   **New Test Suite**: Added `CompositeConnectionTest.java` to verify the routing logic for both reconnection and hardware technology switching (Classic vs. BLE).
*   **Regression Testing**: All tests (100+) passed successfully, ensuring no regressions in the "Single Heartbeat" data flow or thermal analysis algorithms.
*   **Build Integrity**: Verified through the mandatory sequence: `./gradlew spotlessApply test build`.
