# Weather Station v3.7.0 - The User Consent & Hardware Readiness Update

## ✨ New Features & Improvements
*   **Sequential Hardware Readiness**: Implemented a strict, non-overlapping activation sequence for Bluetooth and Location Services. The app now ensures a clean user experience by prompting for Bluetooth first and only alerting for Location Services once Bluetooth is ready.
*   **Interactive Bluetooth Activation**: As per user privacy standards, the app now uses an interactive system prompt (rather than a silent background request) to enable the Bluetooth adapter. 
*   **Enhanced Permission Workflow**: Integrated `BluetoothAdapter.ACTION_REQUEST_ENABLE` into the `PermissionDelegate` lifecycle, providing a unified hardware readiness experience.

## 🛠️ Bug Fixes & Refinements
*   **Verified "Disable Bluetooth on Quit"**: Confirmed that the "Disable Bluetooth on Quit" feature correctly respects user preferences in settings. When enabled, the app now safely shuts down the Bluetooth hardware upon exit to conserve battery.
*   **Refined UI State Management**: Moved Location Service alerts into the `UIEventDelegate` for consistent Snackbar management and state-driven UI updates.

## 🧪 Quality & Tests
*   **Full Suite Verification**: All 110+ unit tests passed, including the connectivity plane and analytical layers.
*   **Build Integrity**: Verified through the mandatory sequence: `./gradlew spotlessApply test build`.
