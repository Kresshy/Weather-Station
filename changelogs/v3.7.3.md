# Weather Station v3.7.3 - The Unified Connection Hardening Update

## 🛠️ Bug Fixes & Improvements
*   **Resolved Redundant Reconnection Loops**: Implemented an "Idempotency Guard" in the `WeatherConnectionController`. This ensures that `startConnection()` is ignored if the system is already in an active state (Connecting, Connected, or Disconnected), preventing duplicate reconnection timers when the app is resumed or rotated.
*   **Consolidated BLE Notifications**: Refined the `BleConnection` callback logic to handle characteristic changes correctly across all Android versions (including 13+). This prevents duplicate data processing and eliminates "empty data" logs.
*   **Defensive Service Management**: Updated `WeatherService` to be more resilient during rapid start/stop cycles, ensuring only one connection plane remains active.

## 🧪 Quality & Tests
*   **State Machine Verification**: Added a new test suite, `WeatherConnectionControllerImplTest.java`, to verify the connection state transitions and the effectiveness of the idempotency guards.
*   **Full Suite Verification**: All 115+ unit tests passed.
*   **Build Integrity**: Verified through the mandatory sequence: `./gradlew spotlessApply test build`.
