# Weather Station v3.8.0 - The Precision & Performance Update

## 🚀 New Features & User Stories
*   **Sliding Window Weather Data (US001)**: Implemented a true time-based sliding window. Historical data older than the selected interval (2, 3, 5, or 10 minutes) is now automatically discarded from the repository to save memory and focus on live monitoring.
*   **Restore File Logging (US002)**: Fully restored and modernized the file logging system. Logs are now written asynchronously to internal storage, bypassing Scoped Storage restrictions and preventing UI thread blocking.
*   **Wind Speed Calibration (US003)**: Refactored the Arduino firmware (v2.1) with a scientifically derived linear calibration formula ($v = f \cdot 0.097 + 0.05$) tailored for custom 4-cup carbon-fiber anemometers.
*   **Initial Point Alignment & Precision (US004)**: 
    *   Updated all UI displays to 2-decimal precision.
    *   Implemented "Bottom-Corner Anchoring" to ensure the first data point of a session starts at the origin of the chart area.
    *   Refactored X-axis to display real-time (HH:mm:ss) labels.

## 🛠️ Bug Fixes & Refinements
*   **Resolved Logging Silence**: Fixed the issue where no log files were being produced on Android 11+ due to storage permission restrictions.
*   **Arduino Overflow Protection**: Upgraded pulse counting to `unsigned int` to prevent counter rollover in high-velocity winds.
*   **GATT & Connection Hardening**: Fixed a NullPointerException in the chart update loop and refined the "Selective Log Export" interface in Settings.
*   **Build Integrity**: Verified through the mandatory sequence: `./gradlew spotlessApply test build`.

## 📦 Deliverables
*   **WeatherStation_v3.8.0_debug.apk**: Signed debug APK with full 3.8.0 feature set.
*   **weatherstation.ino v2.1**: Updated Arduino firmware for custom hardware.
